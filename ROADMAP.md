# OpenBeak Roadmap 🐙🚀

This document outlines the high-level trajectory and vision for **OpenBeak: Macroctopus Agentaculum**.

> [!NOTE]
> Granular feature tracking, milestones, and daily issue triage have been migrated to the **[OpenBeak GitHub Project](https://github.com/codevalve/openbeak/projects)**. 
> Please refer to the live board for current sprint priorities and issue assignments.

---

### Phase 1: The First Tentacle (v1.x) 🐙 
**Goal:** Establish the ultimate stealthy OpenClaw hunter.
- High-performance Go core & Coordinator engine.
- Advanced Hunter tentacles (HTTP detection, CIDR scanning).
- Stealth-first Operator Inking (JSON, activity logs).
- Headless CLI and Tactical TUI support.

### Phase 2: OpenBeak 2.0 Foundation 🎛️
**Goal:** Enhance operator configuration and SIEM integration.
- Interactive TUI Configuration (`openbeak config`).
- Advanced Hunter Tentacles for deeper protocol probing.
- Expanded Ink integrations (Splunk, Elastic).

### Phase 3: The Kraken 🦑
**Goal:** Automated remediation and AI-controlled defense.
- **Beak Tentacles:** Automated remediation (patching configs, revoking permissions).
- **MCP Server Support:** Expose OpenBeak as an MCP server for AI-controlled scanning.
- **Plugin System:** Dynamic loading of external tentacle packages.
